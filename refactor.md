# Refactor

Simplify the code. Make it clearer, shorter, and easier to change.

1. Run the test suite. Establish a green baseline.
2. Identify duplication, unclear names, or unnecessary complexity.
3. Make one change at a time. Run tests after each change.
4. Extract functions when logic is repeated or a block does too much.
5. Rename when a name doesn't say what the thing does.
6. Delete dead code. Remove unused imports, variables, and functions.
7. Run the full test suite. Everything must still pass.

Do not change behavior. Do not add features. Do not add tests for existing untested code — that's a separate task.
